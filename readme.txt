php.ini配置 register_globals=on 否则后台不能正常运行

手动安装方法：
PMA导入一下SQL然后配置一下 admin/config.ini.php 就OK了





register_globals=on就是这个逼玩意弄得，导致很多语法错误，而且打开这个全局参数对系统也不安全，

$_GET["action"]没有初始化就拿来使用就会报notice错误。
可以改为if (isset($_GET["action"]) && $_GET["action"]=="add")

那么你可以通过添加判空函数来解决这个问题，不要直接添加@，这个会导致空白界面。
